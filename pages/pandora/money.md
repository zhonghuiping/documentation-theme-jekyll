## 工作流计费

### 1.消息队列

```
队列数量： 0.9元/个/天

例：假设用户共有 5 个消息队列; 
那么一个月(30 天)的账单金额为: 
0.9【元】 * 5【个】 * 30【天】 = 135 元


数据存储： 0.12元/GB/天

例：
假设日增数据为 20GB;那么 1 个月(30 天)的账单金额为:0.12 【元】* 20【GB】 * 30【天】 * 2【天】 = 144 元;

```

!> 注意：消息队列中每条数据存储 2 天；

### 2.流式计算

```

2U4G        0.432

```


### 3.批量计算

```
资源规格     价格（元/小时）   价格（元/分钟）
1U2G        0.324           0.0054
1U4G        0.54            0.009
2U4G        0.648           0.0108
4U8G        1.3             0.0217
4U16G       2.16            0.036
8U16G       2.56            0.043

例：
假设当前使用了 1U2G(5个)、4U8G(2个)；1U2G(5个)在一个月中共运行了 60 小时 36 分=1476 分钟；
4U8G(2个)在一个月中共运行了 20 小时 12 分=492 分钟；那么 1 个月(30天)的账单金额为：
0.0054【元】 * 5【个】 * 1476【分钟】 + 0.0217【元】 * 2【个】 * 492【分钟】 = 61.2 元；
```

### 5.API调用

```

## 时序数据库

```
数据存储：0.15元/GB/天例：假设日增数据为20GB，存储时限为7天；那么1个月（30天）的账单金额为：0.15【元】 *  20【GB】 *  30【天】 *  7【天】 = 630元；索引值组合：0.001元/个/天例：假设共有索引值组合5000个；那么1个月（30天）的账单金额为： 0.001【元】 *  5000【个】 *  30【天】 = 150元；查询：（function数量 + group by个数 + order by 个数）* 扫描数据量 * 0.0021元/MB例：假设本次查询的 function数量为3、group by数量为1、order by数量为1、扫描数据量为20GB；那么本次查询的费用为：(3+1+1) * 20 * (0.0021 * 1024) = 215.04元；
```
